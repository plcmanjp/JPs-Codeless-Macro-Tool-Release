# v1.5.1

## Improvements

- **Target window close detection during background playback**: If the target window closes while running a macro with background input, the app now clearly notifies you and stops playback. Previously, this could pass silently with no response.
- **Improved stability for screen capture, OCR, and image search**: Improved error handling and responsiveness in the screen capture, OCR (on-screen text recognition) preview, and image search processes.
- **Duplicate shortcut key warning**: Assigning the same shortcut key to two different functions now shows a warning, helping prevent accidental shortcut conflicts.
- **Playback engine unresponsive notice**: If the playback engine rarely becomes unresponsive, the app now guides you to restart it.

## Bug Fixes

- Fixed an issue where the app could fail to start when the favorites file was corrupted.
- Made recording start/stop and playback stop behavior more stable.
- Protected the original file from corruption if a problem occurs while automatically loading the last opened project file.
- Fixed an issue where files could become corrupted when saving while multiple windows were open at the same time.
